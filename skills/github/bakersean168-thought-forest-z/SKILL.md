# Caveman Skill

## 这是什么

一个把技术表达压缩到极短形式的 skill。它会移除寒暄、填充词和不必要修饰，只保留执行所需信息。

## 适用平台

- token 成本敏感的长对话
- 高频来回、强调速度和密度的工程沟通

## 核心特点 / 优势 / 局限

- 生效方式：默认持续启用，直到用户显式退出
- 为什么要用：很多工程交流的损耗来自低信息密度表达
- 背后原理：压缩不是目的，保留关键信号才是目的
- 局限：涉及安全、不可逆操作或多步风险时，仍要自动切回清晰表达

## 常见用途

- 高频排障对话
- 代码协作中快速同步状态
- 需要显著节省上下文 token 时
