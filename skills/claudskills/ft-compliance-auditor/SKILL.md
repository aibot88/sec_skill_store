# 合规审计师

风险识别 / 合规规则匹配 / 金税四期预警应对 / 账务质检 / 整改建议。作为财税处理的最后一道防线，在会计师做完账、税务师算完税之后进行全面检查，确保业财票税四者一致。

## Core Knowledge

### TOP 20 合规风险(按稽查概率排序)

| 序号 | 风险项 | 稽查触发条件 | 后果 |
|---|---|---|---|
| 1 | 虚开发票 | 货物流/资金流/发票流三流不一致 | 刑事责任+补税+罚款 |
| 2 | 隐匿收入 | 银行流水>申报收入，私户收款 | 补税+滞纳金+不缴或少缴税款50%以上5倍以下罚款 |
| 3 | 业财票税不一致 | 合同/发票/付款/入账四者矛盾 | 进项转出+纳税调增 |
| 4 | 私户收款 | 对公账户长期低流水+法人个人账户大额交易 | 隐匿收入认定 |
| 5 | 成本费用虚高 | 毛利率显著低于同行业 | 纳税调增+罚款 |
| 6 | 长期零申报 | 连续3-6个月零申报(各地阈值不同) | 异常户管理+实地核查 |
| 7 | 关联交易转让定价 | 关联交易占比>30% | 特别纳税调整+利息 |
| 8 | 连续亏损不倒闭 | 3年+亏损但正常经营 | 重点稽查对象 |
| 9 | 存货账实不符 | 账面存货与实际严重不符 | 视同销售+补税 |
| 10 | 往来款项长期挂账 | 其他应收款/其他应付款长期不清理 | 视同分红/收入 |

### 金税四期数据比对逻辑

金税四期的核心能力是多数据源交叉比对：

```
银行数据 <-> 增值税发票 <-> 企业所得税申报 <-> 社保人数 <-> 个税申报人数
     |              |               |                |              |
     v              v               v                v              v
  收入验证     进销匹配        利润合理性       用工真实性     薪酬一致性
```

关键比对规则：
- 银行流水收入 vs 增值税申报销售额：偏差>20%触发预警
- 增值税进项行业 vs 企业经营范围：不匹配触发虚开预警
- 社保缴纳人数 vs 个税申报人数：不一致触发用工预警
- 企业所得税利润率 vs 同行业平均：偏差>50%触发预警

### 风险等级评估模型

| 等级 | 条件 | 应对 |
|---|---|---|
| 低风险 | 单一指标轻微异常 | 记录+下期关注 |
| 中风险 | 2个指标异常 或 单一指标严重偏离 | 自查+整改+留存说明 |
| 高风险 | 3+指标异常 或 涉及发票真实性 | 立即整改+准备应对材料 |
| 极高风险 | 涉及虚开/隐匿收入/骗税 | 停止违规行为+聘请专业律师 |

### 月度质检清单(每月16-24日)

```
[ ] 1. 银行余额调节表核对(账实一致)
[ ] 2. 往来款项账龄分析(>1年标记)
[ ] 3. 进销项发票匹配检查
[ ] 4. 费用凭证附件完整性(发票+审批单+合同)
[ ] 5. 工资表与个税申报人数一致性
[ ] 6. 社保缴费基数与工资表一致性
[ ] 7. 固定资产折旧计提完整性
[ ] 8. 收入确认完整性(有无遗漏)
[ ] 9. 印花税应税合同登记完整性
[ ] 10. 税负率同比/环比异常检查
```

## Decision Framework

| 场景 | 检查重点 | 判定标准 |
|---|---|---|
| 新客户首次质检 | 历史账务全面扫描 | 对照TOP 20风险逐项排查 |
| 月度例行质检 | 当月新增业务+指标异常 | 质检清单10项全过 |
| 收到税务预警 | 预警指标定位+数据还原 | 准备情况说明+支撑材料 |
| 汇算清缴前质检 | 全年纳税调整事项 | 12类常见调整项逐一核对 |
| 稽查应对 | 被查年度全部凭证+申报表 | 业财票税四流一致性 |

## Gotchas

1. **金税四期最强的武器不是发票比对，是银行数据** -- 所有对公账户流水税务局都能通过央行/银保监直接调取，不需要企业配合。很多企业以为不开发票就查不到，但银行流水会暴露一切。个人账户大额交易报告标准：现金交易单笔或当日累计5万元以上、非现金(转账)交易单笔或当日累计20万元以上，银行依法向央行反洗钱中心报告。

2. **毛利率异常不一定是问题，但毛利率异常+大额现金收入+库存周转慢=三重预警** -- 单一指标异常可以有合理解释（如行业下行期毛利率下降），但多指标同时异常形成证据链。金税四期的风控模型看的是指标组合，不是单一指标。

3. **连续3年亏损但企业没倒闭，是稽查必查的头号信号** -- 正常商业逻辑下，持续亏损的企业应该倒闭或被股东清算。如果一直亏损还在经营，税务局的合理怀疑是：要么收入隐匿了，要么成本虚增了。必须准备充分的商业合理性解释（如前期投入大、行业特性等）。

4. **长期零申报触发异常户管理** -- 税务局系统对连续零申报的企业自动预警(各地触发阈值不同，常见为连续3-6个月)。后果：限制发票领用、纳税信用降级、可能被列入重点监控或实地核查。即使企业确实没有经营，也应该做合理的费用申报(如房租、基本人工)，避免触发零申报预警。增值税连续3个月零申报即可能触发风控。

5. **关联交易达到一定规模必须准备同期资料** -- 国家税务总局公告2016年第42号规定：年度关联交易金额>2亿(有形资产)或>1亿(其他关联交易)的企业，需要准备本地文档；最终控股企业年度合并收入>55亿的需准备主体文档和国别报告。关联交易占比高(如>30%)是转让定价审查的常见风险指标(非硬性法定阈值)。不按规定准备同期资料=税局可按核定方式调整应纳税所得额。

6. **合规检查不是年底做一次，是每月质检期(16-24日)做** -- 因为问题积累到年底才发现，整改成本成倍增加。月度质检可以在问题萌芽阶段就处理。代账行业的标准工作周期：1-5日采集凭证、6-15日申报、16-24日质检、25-31日准备下月。

## When to Escalate

- 发现可能涉及虚开发票（三流不一致且金额>10万）：刑事风险，立即停止并通知管理层
- 收到税务稽查通知书：需要专业税务律师介入
- 发现前任会计遗留的重大账务错误（影响以前年度纳税）：需要评估是否需要主动更正申报
- 合规检查中发现内控制度缺失（如无审批流程、无岗位分离）：超出账务范畴，需要管理咨询
